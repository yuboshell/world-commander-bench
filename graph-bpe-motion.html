What this is

The question started as “can graph tokenization generate meshes?” The analysis said no — mesh-BPE generation is already published (FreeMesh), and a single SMPL mesh has fixed topology, so graph tokenization of it is degenerate. The promising pivot is human motion: a temporal sequence where BPE could find reusable movement motifs, feeding an LLM generator (Diao's “tokenize a modality for a large model” program; 大模型 = LLM). This page tests the precondition only: is motion BPE-tokenizable? It does not train a generator.

Results at a glance

On real human motion, BPE compresses to 0.13× of the original token length (motion, k=16) vs 0.43× for a shuffled control — a 0.30 gap. The longest recurring movement primitive spans 32 frames. The cyclic highknees clip shows the same: a 14-frame motif BPE reused 3× (the supercut above). Verdict: motion is BPE-tokenizable — the precondition for an LLM motion generator holds. No generator is trained here.

Metrics — what each number means

Compression ratio (CR)
encoded length ÷ original length after BPE. Lower = more compressible = more reusable structure. 1.0 means BPE found nothing to merge.
Shuffled CR (control)
same symbol sequence with frames randomly permuted, then BPE. Destroys temporal order but keeps the symbol multiset. The real-vs-shuffled gap isolates genuine temporal structure from 'BPE compresses anything.'
Longest motif (frames)
the longest contiguous run of frames BPE merged into a single, reused token — i.e. a recurring movement primitive.
k (codebook size)
number of k-means clusters used to discretize each frame's pose into one symbol before BPE. Reported across k to show the result is not an artifact of one setting.

Result 1 — BPE compresses real motion far better than its shuffled control

Each frame's SMPL pose is mapped to one of k k-means symbols; BPE then merges frequent adjacent symbol pairs. We compare the real motion against a control where the same symbols are shuffled in time. If motion has reusable temporal structure, the real sequence should compress much more than the shuffled one.

Result 2 — the merges are multi-frame movement primitives, not noise

A real motif must span several frames and recur. We report the longest BPE token that is reused (≥2 occurrences) at each k.

BPE discovers primitives up to 32 frames long that recur — concrete evidence of reusable motion motifs. The headline demo shows one such token on highknees: a 14-frame (~0.5 s) primitive BPE grouped and reused 3×.

What this de-risks vs what remains

De-risked: human motion carries reusable, multi-frame BPE motifs — robust across codebook size and present in general (not just cyclic) motion. The precondition for BPE-tokenized motion → LLM generation holds, shown on real lab data using only the forward path (no inverse needed). Remaining: (1) a stronger control — an AR(1)/Markov surrogate to separate true recurrence from mere temporal smoothness; (2) scale — more sequences/subjects for higher motif reuse (these clips are ~½–1 min, so top motifs recur only 2–3×); (3) the differentiator — vs MotionGPT (LLM + VQ motion tokens), the thesis must be what BPE buys (reversible, training-free, interpretable, variable-length motifs); (4) confirm Diao's solved “reverse” reconstructs motion exactly; (5) then actually train the generator.
