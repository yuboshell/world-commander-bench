What StarCraft II data exists for RL and LLM work, what it looks like up close, and what Project 1 must synthesize because no corpus supplies it.

Summary. One evening of dataset reconnaissance on yubopc, ahead of Milestone 1. Every corpus that StarCraft II RL work was trained on is language-free, and the LLM-era systems ship no corpus at all — they wrap the live game. The one large static corpus (SC2EGSet, 23,476 tournament games) records every click and unit event but contains, per game, roughly one line of chat. So the pairing P1 needs — (language command, game state) → grounded action — exists nowhere and must be synthesized; that pairing is P1's dataset contribution. The downloaded corpora still earn their disk space, as realism anchors: real state distributions for P3's tokenizer, and a measured human command bandwidth (median ~120, p90 ~150 targeted commands per minute over 2,596 pro player-games) that calibrates the deadline frontier.

2 What existing RL / LLM work actually uses

Grounded in the program's 89-reference survey (survey/command-crowds.csv), the StarCraft II cluster splits into two eras, and neither produced the data P1 needs.

EraRepresentative workData it runs on
RL (2017–2021)AlphaStar; AlphaStar Unplugged~971k anonymized ladder replays (see below) for imitation bootstrapping, then self-play
SMAC; SC2LE / PySC2none — these are environments; data is generated by interaction
LLM (2023–)TextStarCraft II, LLM-PySC2, SwarmBrain, TacticCraft, Adaptive Command, SMAC-Talknone — text wrappers around the live game; no corpus shipped
Near-miss (2019)Waytowich et al., narration-guided RLa small self-built set of command–state pairs; never scaled
Static corpusSC2EGSet (Scientific Data 2023)the one large public corpus — replays + parsed game state, no language channel

"Ladder replay," defined. The ladder is Blizzard's ranked matchmaking queue: players queue for a 1v1, are paired by rating, and move up or down the leagues, as on a chess Elo ladder. Every ladder game auto-records a replay, and Blizzard released anonymized batches of them for research; those millions of ordinary online games are the ladder-replay corpora. The tournament packs downloaded today are the other kind: curated offline pro events — far fewer games, higher and more consistent skill. Ladder buys scale and diversity; tournaments buy quality.

3 Up close A — the pro command stream (raw replay)

One game from the Atlanta pack, parsed with sc2reader: Lambo (Zerg) vs csfalcon (Terran), map "Inside and Out," 10m35s, 15,037 interface events. (Lambo and csfalcon are the players' in-game handles; the parenthetical is each player's race — StarCraft II's three asymmetric factions, Terran / Zerg / Protoss, each with its own units and economy. Matchups are abbreviated by initials: this game is ZvT, and the pvz_task maps in the inventory are Protoss-vs-Zerg. Race fixes the ability vocabulary the executor grounds into.) Lambo issued 1,733 targeted commands (164/min), csfalcon 989 (93/min), plus 2,465 control-group events between them — group addressing is constant, not occasional. A three-second slice of the 5:00 skirmish (the one reviewed in the client), macro-thread lines highlighted:

05.00  Lambo     Attack → (32.8, 35.6)
05.01  csfalcon  Right Click → (23.8, 29.3)
05.02  Lambo     Attack → (32.0, 33.9)
05.05  csfalcon  KD8Charge → (29.9, 27.2)          grenade micro, mid-fight
05.10  Lambo     MorphDrone                            economy, at home
05.11  csfalcon  BuildSupplyDepot → (120.0, 134.0)   base, other corner of the map
05.16  Lambo     SpawnLarva → Hatchery               queen inject cycle
05.18  csfalcon  TrainMarine + ResearchStimpack
  • Fast: several commands per second during fights — the stream, not any single order, is the load.
  • Semantically bare: almost everything is a coordinate click; intent is never written down anywhere.
  • Two threads at once: battle micro on the left of the map interleaves click-by-click with economy at home (highlighted rows). A commander's language would compress each thread differently.
  • The only language in the file: one chat line — "gg :P" at the end.

Pack-wide measurement (2026-07-04; all 1,298 replays, zero parse failures): the median player issues 120 targeted commands per minute (p10 95, p90 149, max 272) — the game above is a 90th-percentile performance, not typical. Zerg commands fastest (median 123/min), Terran slowest (115); winners and losers are nearly identical (122 vs 118), so command volume does not separate them at this level. Control-group events run at ~118/min — group addressing is as frequent as commanding. And the language sparsity now has scale: 0.70 chat lines per 1,000 commands; 34% of player-games contain no chat at all. Median game length 638 s. Full per-player CSV: ~/datasets/sc2/SC2ReSet/atlanta_command_rates.csv (scripts/mine_replay_command_rates.py).

Reviewed sample staged at ~/datasets/sc2/SC2ReSet/samples/2022-Atlanta_Lambo-Z_vs_csfalcon-T_10min.SC2Replay.

4 Up close B — one SC2EGSet record (processed corpus)

Record 0, streamed from the merged JSON via the offsets index (a 2016 game on Lerilak Crest, ~9 minutes). Anatomy, with what each part is good for:

SectionContents (this record)Use for us
gameEvents9,536 events: CameraUpdate 4,501, ControlGroupUpdate 2,240, SelectionDelta 716, Cmd 630interface behaviour at scale — but Cmd is opaque: an integer abilLink, no decoded name or target
trackerEvents653 events: UnitBorn 330, PlayerStats 111 (economy snapshots), UnitInit/Done/Diedthe state channel — evolving world state across 23,476 games; P3 tokenizer material
messageEvents1 event: chat "gg"the language channel, essentially empty — the whole point
metadata, initData, detailsmap, version, anonymized playersstratification and filtering

Practical division of labour between the two corpora: for command-pattern mining, use raw replays + sc2reader (it resolves ability names: "TrainSCV," not abilLink 147); for state distributions at scale, use SC2EGSet's tracker events.

5 The lesson for P1: the record we must synthesize

A professional's intent never passes through language — it goes brain → ~120 clicks per minute (median; p90 ~150). The game's own data can therefore never supply (command, state) pairs, no matter how many replays are parsed. Note also the ratio: one intent ("attack his natural, pull the drones back") explodes into dozens of clicks. That compression is what a language interface buys, and why the executor, not the human, must emit the actions.

Each synthesized P1 record therefore carries five fields:

FieldWhat it isWhere realism comes from
statethe arena / SC2 snapshot when the command is issuedchecked against SC2EGSet tracker-event distributions
commandnatural-language text: scripted canonical intent + LLM paraphrases (the L2 layer)command taxonomy from E1; phrasing variety from paraphrase
ground truthfor concrete commands, an acceptable action set; for operational commands ("harass their mineral line"), an outcome predicate checked on the world after a time window — either way computable because the intent was scripted; only policy / affect commands ("be aggressive") have no predicate and stay human-rated (deferred)by construction
deadlinethe time-to-consequence windowcalibrated by measured pro command bandwidth (median ~120/min, p90 ~150; n=2,596 player-games)
measuredgrounded correctly? within the deadline? or dropped?the harness (E1/E2), unchanged

The corpora downloaded today are not the dataset. They are the realism anchors for the dataset the program generates. One further use, noted 2026-07-03: replays can also yield training-grade language data by back-translation — an LLM captions a burst of clicks with the command that would have produced it. Cheap and abundant, but the caption is inferred rather than known, so it can train an executor and never score one; evaluation ground truth stays scripted-by-construction in the harness.

6 Session notes

  • Replay viewing gotcha: double-clicking a .SC2Replay launches the game binary directly, bypassing the Battle.net launcher's patch step — an out-of-date client then hits "version mismatch" at login. Fix: quit, patch from the launcher, relaunch. Old-version replays fetch their era's assets automatically after login.
  • Review protocol established: text data is reviewed as curated in-line samples with commentary; visual data (replays, videos, rendered figures) is staged with a readable filename and reviewed by eye in its native viewer.
