1.1 Type an order — the interactive demo

The human-commander loop, live: orders are typed into the game's own chat while a melee game of StarCraft II against the built-in AI runs. A 4-billion-parameter model translates each order into a small typed program; a deterministic interpreter binds it against the live game state; two gates sit in front of execution (executability = precondition check, vagueness = clarification request); what passes is executed in the running game. Direct decoding answers the same orders side by side, for comparison only.

1.2 The paradigm

A human commands agents in plain language; a language model turns the command into coordinated action — in real time, on a normal GPU.

1.3 Current status

The current project is the macro command benchmark: StarCraft II with the clock unpaused, scored by execution accuracy — did the executor build, expand, switch tech, and set composition as ordered — not win rate, since units fight on autopilot and win rate measures the game, not the executor. The method we lead with is semantic parsing into a typed program (order→program): a real-time executor should translate, not evaluate — the model parses the order once, and a deterministic interpreter does the state-dependent computation.
