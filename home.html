The paradigm

A human commands agents in plain language; a language model turns the command into coordinated action — in real time, on a normal GPU.

  • Free-form / abstract command — the commander's order in plain language, from concrete ("attack the left flank") to abstract ("hold the line, stay aggressive").
  • Interpreter (LLM) — turns that command into structured intent the agents can act on, fast enough for the live clock and within a memory budget.
  • Coordination (assignment / MARL) — splits the intent across the many agents (who does what), by assignment rules or a learned multi-agent policy.
  • Executor — coarse (discrete actions): issues high-level orders through the game API (move, attack, use an ability); the engine actuates them — the strategy-game domain (e.g. StarCraft II).
  • Executor — medium (trajectories): generates each character's root path — where it goes, a few numbers per frame — and the engine's animation system fills in the limbs. The cheap continuous rung, adjacent to coordination (P4).
  • Executor — fine (continuous motion): generates full-body motion itself, frame by frame (locomotion, gesture, holding formation) — the embodiment domain. All three levels are multi-agent; scale (few → crowd) is a separate axis.

The program grows like a tree: the shared pipeline is the trunk, the executors are its branches, and each task is a leaf. We grow one leaf at a time (the green box marks the one in progress).

Colours follow the tree: the trunk (brown) is the domain-agnostic shared pipeline (the budget wedge sits on the interpreter; coordination is the multi-agent layer); the branches (olive) are the domain-specific executors; the leaf (green) is the task in progress — a box is added for each leaf as it grows; a grey box is a dormant bud, kept in view but not being grown.

Growing now

Leaf 1 — the macro command benchmark (coarse branch): StarCraft II with the clock unpaused. The headline metric is macro command-following fidelity — did the executor build, expand, switch tech, and set army composition as ordered — across the deadline × VRAM frontier. Not win rate: units fight on autopilot, so win rate measures the game, not the executor (the 4B LLM's edge was ~+10 pp and not significant).

  • Milestone 1 (now, yubopc): the SC2 macro-fidelity harness — the LLM executes a fixed scripted strategy under hard drop-late, isolating execution from strategy. No training needed.
  • Milestone 2 (next): one efficiency method (KV-cache eviction / distilled commander / speculative actions) applied to whatever Milestone 1 shows as the binding cost, at equal fidelity — the paper's payload.
  • In reserve — the grid arena (amax): preliminary frontier numbers already exist, so it is not a parallel milestone; it returns as the paper's ablation instrument (clean grounding, command-rate, and memory sweeps) once the SC2 result defines what needs isolating.

Dormant, kept in view: the fine executor (real-time generated motion, long-term) and crowd coordination. The Motion BPE thread runs as a standalone parallel project and becomes the fine executor's on-ramp.

Related work & background

World Commander sits among several research lines; its position is the corner none of them occupy. An 89-paper survey (see Literature) groups the prior work into clusters:

  • LLM agents in real-time games — an LLM issuing commands in a strategy game while the clock runs. HLA (AAMAS 2024), AVA (arXiv 2026), Adaptive Command (arXiv 2025), VideoGameBench (arXiv 2025), AgileThinker (arXiv 2025); AlphaStar (Nature 2019) is the full-RL precedent.
  • Efficient LLM inference under budgets — bounding latency and KV-cache memory at inference time. Speculative Actions (arXiv 2025), Pitfalls of KV Cache Compression (ACL 2026), OBCache (ICML 2026), H2O, SnapKV, StreamingLLM.
  • Vision-language-action models — a slow reasoning backbone feeding a fast action head. π0 (arXiv 2024), OpenVLA (arXiv 2024), Fast-in-Slow (arXiv 2025); NL→StarCraft II grounding (SPIE 2019) and CALVIN (RA-L 2022) for language-conditioned execution.
  • State representation & world models — compressing structured state into fewer tokens per decision. Graph Tokenization (Guo, Diao, ICLR 2026), IRIS / Δ-IRIS (ICLR 2023 / ICML 2024), VQ-VAE (NeurIPS 2017), WHAM (Nature 2025).
  • Crowd & real-time motion generation — text-driven and streaming full-body motion. CrowdMoGen (IJCV 2025), MotionLCM (ECCV 2024), MotionPCM (arXiv 2025), MotionStreamer (ICCV 2025), MotionBricks (SIGGRAPH 2026), InterGen (IJCV 2024). Foundations: T2M / HumanML3D (CVPR 2022, the field's benchmark), MDM (ICLR 2023), T2M-GPT (CVPR 2023), MotionGPT (NeurIPS 2023), MoMask (CVPR 2024).
  • Trajectory prediction & crowd simulation — the middle rung's literature: continuous root paths for many agents. Social Force (Phys. Rev. E 1995), Social-LSTM (CVPR 2016), Social-GAN (CVPR 2018), Trajectron++ (ECCV 2020), MID (CVPR 2022), LED (CVPR 2023, real-time), MotionLM (ICCV 2023, trajectories as tokens), LCTGen (CoRL 2023, language-conditioned).

Across the survey, 52 papers take free-form commands, 23 run in real time, 39 are multi-agent, and 18 measure compute — but none combine all four. The closest five (free-form, real-time, multi-agent) are every one budget-blind. That corner — free-form command of a real-time, multi-agent system under a compute budget — is what the program targets, in both executor domains. In one phrase: efficient machine learning, evaluated in closed-loop games instead of static corpora — the field follows the claim (equal quality at lower cost), and the setting is the one place that claim has never been tested.

Challenges

  • The empty corner. No surveyed paper combines free-form command, real time, many agents, and a compute budget — the program has to build the setting, not just a method.
  • Abstract-intent grounding. Concrete commands ("move to the ridge") are measurable; abstract ones ("be aggressive", "look panicked") have no ground truth — the genuinely novel, hard part. Likely tools: classify-then-act, reward-from-language, or preference data, not supervision. (E4: abstract grounding rose 0.18→0.74 — first evidence it is tractable.)
  • The live clock. Efficiency methods (KV-cache eviction, quantization) are validated only on static text; mid-game, discarding the wrong information changes the outcome, so quality must be scored in-game (grounding / win rate), not by perplexity.
  • No data. The two corpora the program needs — command→coordination and command→crowd-motion — do not exist; both must be synthesized (sim → LLM-paraphrase → render), plus a small human-issued set as the honest test.
  • Coordination at crowd scale. Turning one command into per-agent goals is rule-based at small N but needs a learned policy (RL / MARL) to scale; the O(N²) interaction sim is a known bottleneck.
  • The budget frontier. Holding quality while bounding FPS, end-to-end latency, and VRAM on one consumer GPU — the trade-off the whole program is measured against.

Recent decisions
  • 2026-07-01 — executor axis = level of detail. The two branches are distinguished by the executor's output — coarse discrete actions the engine actuates vs fine continuous motion the model generates — not by "units vs crowd" (a crowd is also units, and scale is a separate axis both share).
  • 2026-07-01 — coarse first. The discrete-action track leads: mature environments, no training needed, clean metrics. The fine track defers to Phase 3.
  • 2026-07-02 — win rate demoted to context. The LLM's SC2 win-rate edge (~+10 pp) was not significant and shrank as n grew: its commands are largely redundant with auto-attack. Macro command-following fidelity is the unconfounded headline.
  • 2026-07-02 — the arena is an instrument, not a benchmark. It appears in the paper as a ground-truth probe and ablation, run once the main result defines what needs isolating — not as a parallel milestone. Effort concentrates on the SC2 macro task and the efficiency methods.
  • 2026-07-02 — phases discarded; dormancy unified. The Phase 1/2/3 numbering is dropped: the program is projects on a build order (benchmarks → methods → the real interface). P3 and P5 are dormant buds (P3 wakes when the benchmark exposes the binding cost; P5 is long-term); P2 stays next, as Milestone 2 is the first paper's payload.
  • 2026-07-02 — trajectories join the axis as the middle rung. From discussion with Xirui Shi and Ji Yang: full-body generation is too costly for games near-term, but a continuous root path is a few numbers per character per frame and the engine animates along it. The detail ladder is now discrete actions → trajectories → full-body motion.
  • 2026-07-02 — Motion BPE stays standalone. Its thesis (what BPE buys over raw VQ tokens: reversible, training-free, variable-length motifs, and a latency/memory win) targets the motion-generation community; it merges with this program only at Phase-3 embodiment.
