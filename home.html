The paradigm

A human commands agents in plain language; a language model turns the command into coordinated action — in real time, on a normal GPU.

  • Free-form / abstract command — the commander's order in plain language, from concrete ("attack the left flank") to abstract ("hold the line, stay aggressive").
  • Interpreter (LLM) — turns that command into structured intent the agents can act on, fast enough for the live clock and within a memory budget.
  • Coordination (assignment / MARL) — splits the intent across the many agents (who does what), by assignment rules or a learned multi-agent policy.
  • Executor — coarse (discrete actions): issues high-level orders through the game API (move, attack, use an ability); the engine actuates them — the strategy-game domain (e.g. StarCraft II).
  • Executor — medium (trajectories): generates each character's root path — where it goes, a few numbers per frame — and the engine's animation system fills in the limbs. The cheap continuous rung, adjacent to coordination (P4).
  • Executor — fine (continuous motion): generates full-body motion itself, frame by frame (locomotion, gesture, holding formation) — the embodiment domain. All three levels are multi-agent; scale (few → crowd) is a separate axis.

The program grows like a tree: the shared pipeline is the trunk, the executors are its branches, and each task is a leaf. We grow one leaf at a time (the green box marks the one in progress).

Growing now

Leaf 1 — the macro command benchmark (coarse branch): StarCraft II with the clock unpaused. Headline metric: macro command-following fidelity (did the executor build, expand, switch tech, and set composition as ordered), not win rate — units fight on autopilot, so win rate measures the game, not the executor.

The sharp thesis the program now stands on: a real-time command executor should translate, not evaluate — the model turns an order into a small typed program, and a deterministic interpreter does the state-dependent computation. This closes the macro gap that model scale, on its own, does not.

  • Milestone 1 (the benchmark) — substantially complete. A command-following harness scored against computable ground truth, built and measured server-side, with one live-loop leg confirming the loop runs end to end. Central finding: micro grounding is solved; macro is a real capability gap; scaling the model buys vocabulary, not the state arithmetic macro needs.
  • Milestone 2 (the methods) — repositioned (2026-07-07). Lead with order→program (translate, don't evaluate) and semantic state-compaction (compress the state representation, not the KV cache). A structure-aware cache press was demoted to a baseline: its token-pinning mechanism is anticipated by prior work, so the contribution is the closed-loop game setting and the translation/compaction architecture, not the press.
  • Next — one rigorous study, not more quick experiments. A difficulty ladder for order→program (direct → derived count → conditional → selective → nested), a strong baseline, generalization tests, and confidence intervals — turning first evidence into a defensible result.

Direction check (how we judge we are productive): goal-driven from the North Star; build a method only for a measured, unsolved difficulty, with novelty checked by reading neighbours' methods, not their abstracts; treat the benchmark and its findings as contributions in their own right, not just the method; and keep in mind that the novel setting — real-time, budgeted, command-following — is itself the contribution, so a higher number on someone else's leaderboard is evidence, not the goal.

Dormant, kept in view: the fine executor (real-time generated motion, long-term) and crowd coordination. The Motion BPE thread runs as a standalone parallel project and becomes the fine executor's on-ramp.

The projects

Each project is one paper-sized unit answering a question its community already values. The program's position is the corner none of the surveyed lines occupy: across the 90-paper survey (Literature), 52 works take free-form commands, 23 run in real time, 39 are multi-agent, and 18 measure compute — none combine all four, and the closest five are budget-blind. In one phrase: efficient machine learning, evaluated in closed-loop games instead of static corpora. Status mirrors the tree above; each project's full plan (objective, hypothesis, design, milestones, risks, venue) is on Plans, and the field map with all clusters stays on Literature.

Program-level challenges (owned by no single project): abstract-intent grounding — concrete commands ("move to the ridge") are measurable, abstract ones ("be aggressive") have no ground truth; likely tools are classify-then-act, reward-from-language, or preference data (first tractability evidence: 0.18 → 0.74 in the E4 v0) — and the budget frontier itself: holding quality while bounding FPS, end-to-end latency, and VRAM on one consumer GPU, the trade-off every project is measured against.
